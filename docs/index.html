Quick Start

  1. Install AUFAIT ERM from Microsoft AppSource.
  2. Create all required SharePoint lists (names must match exactly).
  3. Import and connect the five Power Automate flows.
  4. Configure roles/permissions and branding.
  5. Add the ERM web part to your page and test end-to-end.

Setup Guide (Summary)

  • SharePoint Online tenant & Site Owner permissions
  • No premium Power Automate connectors required
  • Microsoft 365 license with Power Automate capability

Package: SPFx web part (aufait-erms-prod.sppkg), 5 flows (JSON), user & setup guidance, and sample data templates.

Power Automate Flows (Summary)

  1. Mitigation Plan WF – Writes change history to “Mitigation Plan Histories”.
  2. Contingency Plan WF – Writes change history to “Contingency Plan Histories”.
  3. Update Inherent Risk WF – Updates Risk Details; recalculates inherent score.
  4. Update Residual Risk WF – Updates Risk Details; recalculates residual score.
  5. TTL – Risk Initiation WF – Approvals, notifications, reminders, escalations, weekly reports.

Import: Power Automate → My flows → Import (Legacy) → Upload JSON → Map SharePoint/Email connections → Turn on.

Verification Checklist (Summary)

  • 25 lists created correctly; master data populated
  • CompanyCustomization has one active record (branding)
  • All flows imported, connected, and enabled
  • Notifications/emails working; histories recorded
  • Permissions validated by role; web part added

Troubleshooting (Summary)

  • List not found: Check exact names/site URL
  • Permissions: Ensure Contribute on lists
  • Flow failures: Re-authenticate connectors
  • Branding missing: Confirm active CompanyCustomization

Configuration Steps (Detailed)

Step 1: Create All Lists

  1. Go to Settings → Site Contents → New → App.
  2. Create each list/library using the schemas above.
  3. Ensure all column names/types match exactly.

Step 2: Set Up Master Data

  • Populate Projects, Risk Category, Risk Type, Phase, Risk Control Action.
  • Create one active record in CompanyCustomization for branding.

Step 3: Configure User Roles

  • Add admins to M_Admin_Users; ERM heads to M_ERMHead; team members to M_ERMTeam.
  • Configure permissions in M_Roles for each role (View/Create/Edit/Delete/Approve/Export/Reports).

Step 4: Set Up Permissions

  • Grant Contribute permissions to all ERM users across required lists.
  • Optionally use a SharePoint group for ERM users and assign permissions.

Step 5: Add the Web Part

  1. Edit the page → click + → add AUFAIT ERM web part.
  2. Configure settings and publish.

Power Automate Flows (Included)

Included Flow Files

  1. Mitigation Plan WF.json
  2. Contingency Plan WF.json
  3. Update Inherent Risk WF.json
  4. Update Residual Risk WF.json
  5. TTL - Risk Initiation WF.json

Flow Setup Instructions

  1. Import Flows: Go to Power Automate → My flows → Import → Import package (Legacy) → upload JSON files → follow wizard.
  2. Configure Connections: Ensure SharePoint and Email connections are set to your tenant.
  3. Activate: Enable all flows and test with sample data.

Detailed Flow Descriptions

Mitigation Plan WF — Trigger: create/modify in “Mitigation Plan”; writes full history to “Mitigation Plan Histories”, including changes, timestamps, and user.

Contingency Plan WF — Trigger: create/modify in “Contingency Plan”; writes full history to “Contingency Plan Histories”.

Update Inherent Risk WF — Trigger: changes in “Inherent Risk Rating”; updates corresponding “Risk Details” record and recalculates inherent score.

Update Residual Risk WF — Trigger: changes in “Residual Risk Rating”; updates “Risk Details” and recalculates residual score.

TTL - Risk Initiation WF — Triggers include schedule (e.g., daily 9:00 AM) and item changes; sends creation/status/ownership notifications, plan reminders, escalations to ERM Head, weekly summary reports, and rating change notifications.

Flow Configuration Settings

  • Email Templates: Use branding from CompanyCustomization; include risk details and action links.
  • Notification Rules: Immediate (critical), daily digest, weekly summary, escalation for overdue items.
  • Data Validation: Required fields, data consistency, permissions, valid recipient emails.

Monitoring & Maintenance

  • Monitor success rates and set failure alerts.
  • Monthly: review logs, update templates, adjust rules, refresh connections if needed.

Troubleshooting Flows

  • Connection Errors: Re-authenticate SharePoint and Email connectors.
  • Permission Issues: Ensure flows have access to required lists/libraries.
  • Email Delivery: Check spam/transport rules.
  • Data Mapping: Verify field mappings between lists.

Required Permissions for Flows

  • SharePoint: Read/Write across ERM lists
  • Email: Send as configured
  • Power Automate: Standard (no premium connectors)
  • Scheduled trigger permission for reminders/reports

Performance Optimization

  • Batch processing where possible
  • Conditional branches to skip unnecessary actions
  • Error handling and retries
  • Concurrency control and appropriate timeouts

  • Date formats — use standard SharePoint date formats.
  • Branding not applied — verify one active CompanyCustomization record exists.
